Machai MCP Server

Machai MCP Server is a server implementation for the Model Context Protocol (MCP) using the Machai AI framework. It provides integration with GenAI tools and supports both HTTP and STDIO interfaces for AI-powered tool execution and orchestration. The server is designed for extensibility, allowing new tools to be added at runtime through Java SPI discovery without modifying or recompiling the core server code.

Overview

Machai MCP Server exposes Machai AI capabilities to MCP-compatible clients such as Claude Desktop. At startup, it uses FunctionToolsLoader to scan the classpath for SPI-registered tool implementations, configures them through GenericGenaiAdapter, and registers them with the active MCP server instance. The server then listens for incoming requests over the selected transport.

Three transport modes are supported:

  • STDIO — communicates via standard input/output for local process integration. This is the default mode when no port is specified.
  • Stateless HTTP — each request is handled independently with no persistent session state, using HttpServletStatelessServerTransport.
  • Streamable HTTP — supports incremental result delivery over a persistent HTTP connection, using HttpServletStreamableServerTransportProvider. Selected with the -s flag alongside -p.

Package structure

org.machanism.machai.mcp.server

Contains all server entry points, transport implementations, tool registration infrastructure, and shared base classes.

  • McpServer — main entry point. Parses command-line arguments and launches the appropriate server instance in STDIO or HTTP mode.
  • StdioMcpServer — configures and starts the server over standard input/output for local process-based integration.
  • HttpStatelessMcpServer — configures and starts a stateless HTTP MCP server backed by Jetty.
  • HttpStreamableMcpServer — configures and starts a streamable HTTP MCP server backed by Jetty.
  • AbstractHttpMcpServer — shared base for HTTP server variants; manages the Jetty server lifecycle and port binding.
  • AbstractMcpServer — top-level base class; defines the tools() and start() contracts and the ToolSpecificationBuilder interface.
  • GenericGenaiAdapter — adapts SPI-loaded function tools into transport-specific MCP tool specifications using a provided ToolSpecificationBuilder.

Command-line options

When running McpServer directly:

  • -h, --help: Show help message and exit.
  • -d, --projectDir: Specify the project directory path.
  • -n, --name: MCP server name reported to clients (default: mcp-machai-server).
  • -v, --version: MCP server version reported to clients (default: implementation version or latest).
  • -p, --port: Port for HTTP mode. If omitted, the server runs in STDIO mode.
  • -s, --session: Use streamable HTTP transport instead of stateless (only applies with -p).

Extensibility

New tools are added by implementing the Machai Functional Tools SPI and packaging the implementation with a META-INF/services entry in the JAR. The server discovers and registers them automatically at startup via FunctionToolsLoader without any changes to the server itself.

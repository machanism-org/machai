Bindex Maven Plugin

Bindex Maven Plugin enables automated generation and registration of bindex metadata for Maven projects. It facilitates library discovery, integration, and assembly by leveraging structured metadata and GenAI-powered semantic search within the Machanism ecosystem.

Overview

This module is a Maven plugin (a set of Mojos) that integrates Machai/Bindex operations into a Maven build. Each goal runs in the context of the current Maven module and typically skips projects with pom packaging (aggregator/parent modules), since they do not produce an artifact to index.

Architecture at a glance

The Mojos in this plugin delegate to the underlying Bindex services provided by bindex-core:

  • org.machanism.machai.bindex.BindexCreator – create or refresh a local Bindex index for the current module.
  • org.machanism.machai.bindex.BindexRegister – scan a project and publish metadata to a registry.

Packages

  • org.machanism.machai.bindex.maven – Maven plugin goals (Mojos) that expose Bindex operations. The goals share common behavior through an abstract base Mojo and execute against the current org.apache.maven.project.MavenProject.

Goals and typical usage

  • create – create a new Bindex index for the current module.
  • update – refresh an existing Bindex index.
  • register – scan the current project and publish metadata to a registry.
  • clean – remove plugin-generated temporary artifacts.
mvn org.machanism.machai:bindex-maven-plugin:create -Dbindex.genai=OpenAI:gpt-5
mvn org.machanism.machai:bindex-maven-plugin:update -Dbindex.genai=OpenAI:gpt-5
mvn org.machanism.machai:bindex-maven-plugin:register -Dbindex.genai=OpenAI:gpt-5 -Dbindex.register.url=http://localhost:8080
mvn org.machanism.machai:bindex-maven-plugin:clean

Configuration model

The GenAI provider/model is selected via the system property -Dbindex.genai (for example, OpenAI:gpt-5). The register goal additionally uses -Dbindex.register.url to choose the registry endpoint.

Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis.

The application processes source code, documentation, project site content, and other repository files directly from the filesystem without requiring a project build step.

Purpose

Ghostwriter is a filesystem-oriented documentation and prompt-execution tool for repositories that embed author instructions inside project files. It discovers @guidance directives, gathers project layout metadata, builds repository-aware prompts, and applies generated output back to matching files.

In addition to guidance-driven updates, Ghostwriter supports reusable Act templates that define structured prompt workflows for repeated maintenance, review, and content-generation tasks.

Architecture overview

The runtime is organized around a small set of cooperating packages. A command-line processor selects scan targets, traverses the project layout, delegates file inspection to format-aware reviewers, composes prompts, and executes them through a configured {@code org.machanism.machai.ai.provider.Genai} provider. Optional function tools extend prompt execution with controlled access to project files, command execution, web access, and Act lifecycle operations.

Package overview

org.machanism.machai.gw.processor

This package contains the Ghostwriter command-line entry point and the core processing pipeline. It is responsible for traversing repositories, recognizing module structure, applying include and exclude rules, collecting project metadata, constructing prompts, and invoking the configured AI provider.

{@code org.machanism.machai.gw.processor.AbstractFileProcessor} provides shared traversal, path filtering, matcher handling, and multi-module support. {@code org.machanism.machai.gw.processor.AIFileProcessor} builds on that foundation by initializing provider access, assembling request content, logging request inputs when configured, and registering runtime tools. Concrete implementations divide execution into guidance-driven processing through {@code org.machanism.machai.gw.processor.GuidanceProcessor} and Act-driven processing through {@code org.machanism.machai.gw.processor.ActProcessor}. Supporting types such as {@code org.machanism.machai.gw.processor.Ghostwriter}, {@code org.machanism.machai.gw.processor.GWConstants}, and {@code org.machanism.machai.gw.processor.ActNotFound} complete the package by handling bootstrap, configuration constants, and error reporting.

org.machanism.machai.gw.reviewer

This package provides reviewer components that inspect project files for embedded @guidance directives and convert those directives into normalized prompt fragments. The central extension point is {@code org.machanism.machai.gw.reviewer.Reviewer}, a service-provider interface implemented by file-format- specific reviewers.

Reviewers understand the syntax of their target formats and extract guidance from locations such as Java comments, Markdown content, HTML comments, Python comments, TypeScript comments, PlantUML sources, and dedicated text guidance files. The package includes specialized handling for Java source and {@code package-info.java} files through {@code org.machanism.machai.gw.reviewer.JavaReviewer}, as well as standalone guidance files through {@code org.machanism.machai.gw.reviewer.TextReviewer}. Reviewers also compute stable project-relative paths using the detected project layout so downstream prompts can refer to files consistently.

org.machanism.machai.gw.tools

This package supplies host-integrated function tools that can be exposed to the AI provider during prompt execution. These tools let Ghostwriter workflows interact with the surrounding environment in controlled and bounded ways.

The package includes Act-management utilities, file-system operations, command execution support, web access, security checks for commands, bounded output buffering, and control-flow exceptions used to switch workflow episodes. Together, these types enable repository-aware automation while keeping environment access explicit and host-configured.

Processing workflow

  1. Select scan targets using directories or path-matcher expressions.
  2. Traverse the project layout across source, test, documentation, and module boundaries.
  3. Inspect files with format-specific reviewers to locate embedded guidance.
  4. Compose prompts from project metadata, extracted instructions, and optional Act templates.
  5. Register tools that expose file, command, web, and Act operations when enabled.
  6. Execute requests through the configured GenAI provider and apply generated results to files.

Usage notes

Ghostwriter operates directly on repository content rather than compiled project models. It is designed for documentation generation, guided maintenance, and structured AI-assisted updates where instructions live next to the files they govern.

A typical invocation starts the CLI entry point and provides a scan directory or matcher expression, for example {@code java -jar gw.jar [options]}. Matching targets can be repository-relative paths such as {@code src/main} or matcher expressions supported by {@code java.nio.file.FileSystems#getPathMatcher(String)}.

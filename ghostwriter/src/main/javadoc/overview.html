Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis. It operates as a filesystem-first command-line utility and supports all types of project files7including source code, documentation, project site content, and other relevant files.

Purpose and scope

Ghostwriter walks a project directory tree, discovers embedded @guidance: instructions, and orchestrates the creation of provider-ready prompts for a configured GenAI provider.

  • Filesystem-only processing: scanning does not compile code and does not resolve dependencies.
  • Child-first traversal: when module directories are present, they are processed before the parent.
  • Format-aware extraction: guidance is extracted by file-type reviewers (for example Java, Markdown, and HTML).

Key concepts

Guidance
A mandatory @guidance: block embedded in a file comment. Guidance is treated as file-scoped instructions that can be extracted and included in a prompt.
Reviewer
A file-format-aware scanner that locates and extracts @guidance: blocks according to the comment conventions of a specific format.
Processor pipeline
The orchestration layer that traverses the filesystem, selects a reviewer per file, composes prompts from guidance and project context, and dispatches the prompts to the configured GenAI provider.

High-level architecture

Ghostwriter is organized into two primary packages: the processing pipeline and the reviewer SPI with format-specific implementations.

Package overview

org.machanism.machai.gw.processor

Project scanning and prompt orchestration for the Ghostwriter command-line tool. The processors in this package implement a filesystem-based pipeline that:

  • traverses a project directory tree (including multi-module layouts),
  • selects scan targets via explicit directories or glob:/regex: matchers,
  • extracts embedded @guidance: directives via extension-specific reviewers, and
  • builds and dispatches prompts to a configured GenAI provider.

Processing operates strictly on the filesystem: projects are not built and dependencies are not resolved during scanning. When module directories are present, they are processed child-first so that module content can be handled before any parent-level aggregation.

org.machanism.machai.gw.reviewer

File-format-aware scanners used by Ghostwriter to discover embedded @guidance instructions. This package defines the reviewer SPI and includes concrete implementations for common project artifacts such as Java, TypeScript, Python, Markdown, HTML/XML, PlantUML, and plain-text files.

A reviewer inspects an individual file, detects whether it contains the guidance tag, and if present produces a normalized prompt fragment (typically including the project-relative file path and relevant content) for consumption by the processing pipeline.

Processing model

Scan targets and matchers

A scan target can be a directory path (typically relative to the project root) or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

  • Directory path (for example, src/main)
  • glob: patterns (for example, glob:**/*.java)
  • regex: patterns (for example, regex:^.*/[^/]+\\.java$)

Guidance extraction and orchestration

For each selected file, Ghostwriter chooses an appropriate reviewer based on the file format, extracts any embedded guidance blocks, and combines them with project context (such as the project-relative file path and relevant content) to build a provider-ready prompt.

Typical usage

java -jar gw.jar <scanTarget> [options]

The <scanTarget> argument can be a relative directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

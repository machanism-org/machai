Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis.

Overview

Ghostwriter is a filesystem-first documentation and content generation tool. It traverses a project directory tree, discovers embedded @guidance: instructions in supported files, and orchestrates provider-ready prompts for a configured GenAI provider. The resulting content is applied in-place to the same files.

Ghostwriter supports all types of project files—including source code, documentation, project site content, and other relevant files.

High-level architecture

At a high level, Ghostwriter consists of:

  • Processors that traverse the filesystem, select scan targets, compose prompts, and apply results.
  • Reviewers (an SPI) that extract guidance blocks from specific file formats and normalize them into prompt fragments.
  • Acts (TOML templates) that provide reusable prompt/instruction bundles for common workflows.
  • Provider integration via {@code org.machanism.machai.ai.manager.Genai} for prompt dispatch and tool registration.

Packages

org.machanism.machai.gw.processor

Project scanning, guidance extraction, and prompt orchestration for the Ghostwriter command-line tool. This package contains the CLI entry point and the processors responsible for scanning a project directory tree, extracting embedded @guidance: directives, and invoking the configured GenAI provider.

Processing is strictly filesystem-based: projects are not built and dependencies are not resolved during scanning. When a path matcher is used (via glob: or regex:), inclusion decisions are made against project-relative paths and known build/tooling directories are ignored.

org.machanism.machai.gw.reviewer

File-format-aware scanners that locate embedded @guidance instructions in project files and convert them into normalized prompt fragments for Ghostwriter's downstream processing pipeline.

The central abstraction is the Reviewer service-provider interface (SPI). Each reviewer targets a specific file type (for example Java sources, Markdown, HTML/XML, TypeScript, Python, PlantUML, or plain-text guidance files), understands that format's comment conventions, and returns a formatted fragment that can be assembled into a single request for the LLM.

org.machanism.machai.gw.tools

GenAI provider tool registration for Ghostwriter-specific operations. This package exposes selected project capabilities (for example listing and loading Act templates) as callable tools.

Processing workflow

  1. Select scan targets (directory roots and/or matchers such as glob: and regex:).
  2. Traverse the filesystem and select files to process using include/exclude rules.
  3. Extract guidance with a format-specific reviewer.
  4. Compose prompts using extracted guidance, acts/templates, and project metadata.
  5. Dispatch the request to the configured GenAI provider and apply the result.

Typical usage

java -jar gw.jar <scanDir> [options]

The <scanDir> argument may be a project-relative directory (for example, src/main) or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher (for example, glob:**/*.java).

Ghostwriter

Ghostwriter is a command-line utility that scans a project’s filesystem, extracts embedded @guidance: directives from supported file formats, and orchestrates GenAI-driven updates to the files it finds.

Ghostwriter is filesystem-based: it does not compile sources, execute builds, or resolve dependencies. For multi-module layouts, modules are processed child-first so module guidance is applied before the parent project.

Architecture

The codebase is organized around two cooperating packages:

org.machanism.machai.gw.processor
Command-line entry point and the scan → extract → compose → dispatch pipeline. This package is responsible for traversing project folders (including child-first module traversal), applying include/exclude rules, composing prompts, and invoking a configured GenAI provider.
org.machanism.machai.gw.reviewer
Reviewer SPI and format-specific implementations used to extract and normalize guidance from project files. Reviewers are loaded via Java’s ServiceLoader and mapped to file extensions.

org.machanism.machai.gw.processor

This package contains the user-facing CLI and the processing pipeline that turns project files into GenAI prompt runs.

  • Ghostwriter — CLI entry point. Loads configuration, parses options (scan target, root, excludes, instructions, default guidance, threading, input logging), and invokes the processing pipeline.
  • FileProcessor — traverses the filesystem, discovers modules (child-first), delegates file parsing to reviewers, assembles prompts, and dispatches them to the selected GenAI provider.

Key behaviors

  • Child-first module traversal: modules are processed before their parent directory.
  • No dependency resolution: the processor does not build projects or resolve artifacts.
  • Configurable scope: the scan target may be a directory path or a java.nio.file.FileSystems.getPathMatcher expression (for example, glob:**/*.java or regex:...).
  • Optional auditing: composed prompt inputs may be logged to a temporary directory for debugging and traceability.

Prompt composition (high-level)

For each processed file, Ghostwriter composes a prompt using:

  1. System instructions (configured or supplied via the CLI).
  2. Environment constraints (for example, OS-specific command syntax rules).
  3. Project context (layout, source/test/doc roots, modules).
  4. Per-file extracted guidance (or default guidance if the file contains none).

org.machanism.machai.gw.reviewer

The reviewer package defines the Reviewer service-provider interface (SPI) and a set of implementations responsible for extracting embedded @guidance instructions.

Reviewers are designed to understand a format’s comment conventions and return a normalized prompt fragment for downstream processing.

Implementations in this project include reviewers for:

  • Java source (.java)
  • Markdown (.md)
  • HTML (.html)
  • TypeScript (.ts)
  • Python (.py)
  • PlantUML (.puml)
  • Plain text guidance (.txt)

Typical usage

java -jar gw.jar <scanDir> [options]

The <scanDir> argument can be a relative directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

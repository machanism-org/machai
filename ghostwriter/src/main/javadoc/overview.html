Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis.

Ghostwriter supports all types of project files—including source code, documentation, project site content, and other relevant files.

Purpose and scope

Ghostwriter is a filesystem-first tool that traverses a project directory tree, discovers embedded @guidance: instructions, and orchestrates the creation of provider-ready prompts for a configured GenAI provider.

  • Filesystem-based processing: scans files as they exist on disk; it does not build projects.
  • Child-first processing: when processing multi-module layouts, child modules are processed before parent aggregation.
  • Format-aware guidance extraction: guidance is extracted via file-format reviewers.

High-level architecture

The implementation is organized around an orchestration pipeline (project traversal, matching, prompt composition, and dispatch) and a reviewer SPI (format-specific guidance extractors).

Core workflow

  1. Select scan targets (directory roots and/or path matchers such as glob: and regex:).
  2. Traverse the filesystem and select files to process using include/exclude rules.
  3. Extract guidance from file comments using a format-specific reviewer.
  4. Compose prompts using project metadata and extracted guidance.
  5. Dispatch the prompt to the configured GenAI provider and apply the result (for example, updated documentation).

Packages

Ghostwriter is organized around two core packages:

org.machanism.machai.gw.processor

Project scanning, guidance extraction, and prompt orchestration for the Ghostwriter command-line tool. This package contains the CLI entry point and processors responsible for scanning a project directory tree, extracting embedded @guidance: directives, and invoking the configured GenAI provider. Processing is strictly filesystem-based (projects are not built and dependencies are not resolved). For multi-module layouts, processing is performed child-first.

org.machanism.machai.gw.reviewer

File-format-aware scanners used by Ghostwriter to discover embedded @guidance instructions and convert them into normalized prompt fragments for downstream processing. Implementations scan formats such as Java, Markdown, HTML/XML, TypeScript, Python, PlantUML, and plain-text guidance files.

Typical usage

java -jar gw.jar <scanTarget> [options]

The <scanTarget> argument can be a relative directory path (for example, src/main) or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher (for example, glob:**/*.java).

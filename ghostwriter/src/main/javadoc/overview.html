Ghostwriter

Ghostwriter is a command-line utility that scans a project’s filesystem, extracts embedded @guidance: directives from supported file formats, and orchestrates GenAI-driven updates. The tool is intentionally filesystem-based: it does not compile sources, execute builds, or resolve dependencies.

Purpose and scope

Ghostwriter turns in-repository guidance (for example, @guidance: comments in source, documentation, and site content) into provider-ready prompts that can be used to update project files. Its scan-and-review model is designed to work across the full set of files that commonly make up a project: source code, documentation, project site content, and other relevant artifacts.

Architecture

The implementation is organized into two cooperating packages:

org.machanism.machai.gw.processor
Command-line entry point and processing pipeline responsible for filesystem traversal, prompt composition, and provider orchestration. The processor walks a project tree, applies include/exclude rules, extracts guidance via reviewers, and submits the composed prompt to a configured GenAI provider. In multi-module layouts, modules are processed child-first (each module is scanned before the parent directory).
org.machanism.machai.gw.reviewer
Reviewer service-provider interface (SPI) and format-specific implementations used to extract and normalize guidance from project files. Reviewers understand comment conventions for a specific format, find @guidance: blocks, and return normalized guidance fragments for downstream prompt composition.

Processing model

Filesystem-first scanning

Ghostwriter operates directly on files and directories. It does not compile the project, execute builds, or resolve dependencies; the guidance pipeline is based on file content and project layout only.

Module traversal

When a project layout declares modules, Ghostwriter processes modules before their parent directory (child-first). This helps ensure module-local guidance is discovered and applied before parent-level guidance.

Scan targeting

A scan target can be a directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

  • Directory path (typically relative to the project root)
  • glob: patterns (for example, glob:**/*.java)
  • regex: patterns (for example, regex:^.*/[^/]+\\.java$)

Guidance extraction and orchestration

For each selected file, Ghostwriter delegates guidance extraction to a matching reviewer. The extracted guidance is normalized (including project-relative path context) and combined with project context to form a provider-ready prompt.

Packages

org.machanism.machai.gw.processor

Contains the user-facing CLI and the processing pipeline that turns scanned files into GenAI provider calls.

  • Filesystem traversal: walk the project tree, support include matchers (glob:/regex:), and apply exclusion rules.
  • Provider orchestration: create and configure a provider, apply any configured function tools, and optionally log composed prompt inputs for traceability.
  • Guidance processing: select reviewers by file extension, extract embedded guidance, compose prompts, and dispatch them to the provider.

org.machanism.machai.gw.reviewer

Defines the Reviewer SPI and contains implementations responsible for extracting embedded @guidance directives from different file formats.

Reviewers commonly cover both code and documentation formats, such as Java source (including package-info.java), Markdown, HTML/XML, TypeScript, Python, PlantUML, and plain text guidance files.

Typical usage

java -jar gw.jar <scanTarget> [options]

The <scanTarget> argument can be a relative directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

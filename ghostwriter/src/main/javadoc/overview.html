Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis.

Purpose

Ghostwriter is a filesystem-first documentation and content generation tool. It traverses a project directory tree, discovers embedded @guidance: instructions in supported files, composes provider-ready prompts, and applies generated output back to the original files.

Ghostwriter supports all types of project files—including source code, documentation, project site content, and other relevant files.

High-level architecture

The processing pipeline follows a consistent sequence:

  • Select scan roots and matching rules.
  • Traverse the filesystem and choose files to process.
  • Extract guidance blocks using a file-format-aware reviewer.
  • Compose prompts (optionally using reusable Act templates).
  • Dispatch requests via {@code org.machanism.machai.ai.manager.Genai} and apply results in-place.

Package overview

org.machanism.machai.gw.processor

Command-line entry point and processing pipeline. This package provides the Ghostwriter CLI and the processors that scan a project directory, select files, and orchestrate prompt execution against a configured {@code org.machanism.machai.ai.manager.Genai} provider.

Key concepts include:

  • Project layout: scanning behavior can be derived from {@code org.machanism.machai.project.layout.ProjectLayout} (for example, identifying source/test/documentation roots).
  • File traversal and filtering: shared traversal logic and common exclusions are centralized in {@code org.machanism.machai.gw.processor.AbstractFileProcessor}.
  • Provider execution: prompt composition and provider invocation are handled by {@code org.machanism.machai.gw.processor.AIFileProcessor}.

The CLI typically runs in one of two modes: Guidance (extract inline @guidance: directives and build a prompt from them) or Act (apply a TOML-defined Act template to matching files).

org.machanism.machai.gw.reviewer

File-format-aware components that locate and extract embedded {@code @guidance} directives and convert them into normalized prompt fragments. The package is organized around the {@code org.machanism.machai.gw.reviewer.Reviewer} interface, with implementations for multiple formats (Java, Markdown, HTML/XML, TypeScript, Python, PlantUML, and plain text).

Reviewers generally compute a stable project-relative path via {@code org.machanism.machai.project.layout.ProjectLayout#getRelativePath(java.io.File, java.io.File)} and then format prompt fragments using templates from the {@code document-prompts} resource bundle.

org.machanism.machai.gw.tools

GenAI provider tool registration for Ghostwriter-specific operations. This package exposes selected Ghostwriter capabilities as callable tools available to the configured provider.

Processing workflow

  1. Select scan targets (directory roots and/or matchers such as glob: and regex:).
  2. Traverse the filesystem and select files using include/exclude rules.
  3. Extract guidance with a format-specific reviewer.
  4. Compose prompts using extracted guidance and optional Act templates.
  5. Dispatch the request to the configured GenAI provider and apply the result.

Typical usage

{@code java -jar gw.jar [options]}

The {@code } argument may be a project-relative directory (for example, {@code src/main}) or a matcher expression supported by {@code java.nio.file.FileSystems#getPathMatcher(String)} (for example, {@code glob:**/*.java}).

Ghostwriter

Ghostwriter is an advanced documentation engine that automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis.

Overview

Ghostwriter is a filesystem-first documentation and content generation tool. It traverses a project directory tree, discovers embedded @guidance: instructions in supported files, composes provider-ready prompts, and applies generated output in-place to the original files.

Ghostwriter supports all types of project files—including source code, documentation, project site content, and other relevant files.

High-level architecture

The core flow is:

  • Select scan roots and matching rules.
  • Traverse the filesystem and choose files to process.
  • Extract guidance blocks using a file-format-aware reviewer.
  • Compose prompts (optionally using reusable Act templates).
  • Dispatch requests via {@code org.machanism.machai.ai.manager.Genai} and apply results in-place.

Packages

org.machanism.machai.gw.processor

Command-line entry point and processing pipeline responsible for scanning a project directory, selecting files, extracting guidance, and orchestrating prompt execution against a configured {@code org.machanism.machai.ai.manager.Genai} provider.

Key responsibilities include:

  • Discover project and module structure via {@code org.machanism.machai.project.layout.ProjectLayout}.
  • Traverse the filesystem and apply include/exclude rules to determine which paths are processed.
  • Support execution modes:
    • Guidance mode: extract embedded {@code @guidance:} directives using {@code org.machanism.machai.gw.reviewer.Reviewer} implementations and build prompts from those directives.
    • Act mode: load TOML-based Act templates and execute them across matching files.
  • Invoke the provider and (optionally) log composed request inputs.

org.machanism.machai.gw.reviewer

File-format-aware components that extract embedded {@code @guidance} instructions and convert them into normalized prompt fragments.

The primary abstraction is the {@code Reviewer} service-provider interface (SPI). Each implementation targets one or more file types, locates the {@code @guidance} tag according to the format's comment conventions, and emits a formatted fragment suitable for inclusion in a single request to the LLM.

Reviewers typically read files as UTF-8, compute project-relative paths via {@code org.machanism.machai.project.layout.ProjectLayout#getRelativePath(java.io.File, java.io.File)}, and format prompt fragments using templates from the {@code document-prompts} resource bundle.

org.machanism.machai.gw.tools

GenAI provider tool registration for Ghostwriter-specific operations.

This package exposes selected Ghostwriter capabilities (for example listing and loading Act templates) as callable tools available to the configured provider.

Processing workflow

  1. Select scan targets (directory roots and/or matchers such as glob: and regex:).
  2. Traverse the filesystem and select files using include/exclude rules.
  3. Extract guidance with a format-specific reviewer.
  4. Compose prompts using extracted guidance and optional Act templates.
  5. Dispatch the request to the configured GenAI provider and apply the result.

Typical usage

{@code java -jar gw.jar [options]}

The {@code } argument may be a project-relative directory (for example, {@code src/main}) or a matcher expression supported by {@code java.nio.file.FileSystems#getPathMatcher(String)} (for example, {@code glob:**/*.java}).

Ghostwriter

Ghostwriter is a command-line utility that scans a project’s filesystem, extracts embedded @guidance: directives from supported file formats, and orchestrates GenAI-driven updates. The tool is intentionally filesystem-based: it does not compile sources, execute builds, or resolve dependencies.

Architecture

Ghostwriter’s pipeline is split across two cooperating packages:

org.machanism.machai.gw.processor
The CLI entry point and the scan → extract → compose → dispatch pipeline. It traverses the project tree, applies include/exclude rules, composes prompts from project context and extracted guidance, and invokes a configured GenAI provider. For multi-module builds, modules are processed child-first (each module is scanned before the parent project directory).
org.machanism.machai.gw.reviewer
The reviewer SPI and format-specific implementations used to extract and normalize guidance from project files. Reviewers are typically loaded via Java’s ServiceLoader and are selected by file extension.

Processing model

Ghostwriter operates on project files directly. It can scan and extract guidance from many file types, including source code, documentation, project site content, and other relevant project files.

Module traversal

When a project layout declares modules, Ghostwriter processes modules before their parent directory (child-first). This ensures module-local guidance is applied before parent-level guidance.

Scan targeting

The scan target can be either a directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

  • Directory path (typically relative to the project root)
  • glob: patterns (for example, glob:**/*.java)
  • regex: patterns (for example, regex:^.*/[^/]+\\.java$)

org.machanism.machai.gw.processor

The processor package contains the user-facing CLI and the processing pipeline that turns project files into GenAI prompt runs.

  • Ghostwriter — CLI entry point. Loads configuration, parses options, and invokes the processing pipeline.
  • GuidanceProcessor — Walks the filesystem (including child-first module traversal), delegates file parsing to reviewers, assembles prompts, and dispatches them to the selected provider.

Key behaviors

  • Child-first module traversal: modules are processed before the parent directory.
  • No dependency resolution: projects are not built and dependencies are not resolved during scanning.
  • Configurable scan scope: scanning can target directories or path-matcher expressions.
  • Optional auditing: composed prompt inputs can be logged for debugging and traceability.

org.machanism.machai.gw.reviewer

The reviewer package defines the Reviewer service-provider interface (SPI) and includes implementations responsible for extracting embedded @guidance instructions from various file formats.

Each reviewer understands a format’s comment conventions and returns a normalized guidance fragment for downstream prompt composition.

Included reviewers

  • Java source (.java, including package-info.java)
  • Markdown (.md)
  • HTML (.html)
  • TypeScript (.ts)
  • Python (.py)
  • PlantUML (.puml)
  • Plain text guidance (.txt)

Typical usage

java -jar gw.jar <scanDir> [options]

The <scanDir> argument can be a relative directory path or a matcher expression supported by java.nio.file.FileSystems.getPathMatcher.

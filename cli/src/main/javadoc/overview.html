Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Overview

This module provides a Spring Shell-based command-line interface (CLI) for the Machai toolchain. It is designed for both interactive usage (REPL) and scripted execution in automation and CI pipelines.

High-level architecture

  • Application entry point: org.machanism.machai.cli.MachaiCLI bootstraps the application and starts the Spring Shell runtime.
  • Command components: A set of Spring Shell command classes implement user-facing operations such as configuration management, bindex generation and registration, semantic search (“pick”), assembly, document processing, and cleanup.

Package overview

Public functionality is concentrated in a single package:

  • org.machanism.machai.cli – Spring Shell-based CLI for Machai.
    • Contains the Spring Boot entry point and Spring Shell command components that expose Machai workflows.
    • Supports interactive REPL usage as well as one-shot command execution.
    • Command groups include configuration, indexing/registration, picking/assembly, document processing (including “Act mode”), and cleanup.

Key types

  • MachaiCLI – Application entry point; initializes Spring Shell.
  • ConfigCommand – Persists and prints defaults (for example, working directory and GenAI provider/model).
  • BindexCommand – Generates and registers bindex metadata.
  • AssembyCommand – Performs semantic search (“pick”) and assembles an output project.
  • GWCommand – Scans folders and processes documents/files using the Ghostwriter pipeline.
  • ActCommand – Runs a predefined interactive action/prompt (“Act mode”).
  • CleanCommand – Removes Machai temporary artifacts (for example, .machai).

Typical usage

config model OpenAI:gpt-5.1
config root .\\my-project
bindex --dir .\\my-project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out
gw --scanDirs .\\my-project --excludes target,.git
act --action "Summarize the architecture"
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows; use the clean command to remove it when needed.
  • Command names and options are defined by Spring Shell components in org.machanism.machai.cli. Consult each command class’ Javadoc for detailed semantics and option defaults.

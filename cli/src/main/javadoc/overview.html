Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Purpose and scope

This module produces an executable, Spring Shell-based command-line interface for Machai workflows. It supports both interactive usage (REPL) and non-interactive invocation.

Architecture at a glance

  • Entry point: org.machanism.machai.cli.MachaiCLI bootstraps Spring Shell.
  • Configuration loading: optional system properties are loaded from machai.properties, or from the file specified via -Dconfig=....
  • Commands: Spring Shell command components expose configuration management, bindex operations, assembly workflows, document processing, and cleanup.

Package overview

The public API surface for this module is concentrated in a single package:

  • org.machanism.machai.cli – Spring Shell-based command-line interface (CLI) for Machai.
    • Bootstraps the CLI application and loads configuration (for example, from machai.properties).
    • Manages persistent CLI defaults (for example, working directory and GenAI provider/model settings).
    • Executes commands for bindex generation/registration, semantic search and assembly, document processing, and cleanup of Machai temporary directories.

Key types

  • MachaiCLI – Boots the application and loads optional system properties.
  • ConfigCommand – Manages persistent defaults such as working directory (dir), GenAI provider/model (genai), and semantic-search score threshold (score).
  • AssemblyCommand – Performs library picking (semantic search) and assembles an application skeleton using a configured GenAI provider.
  • BindexCommand – Generates bindex metadata for projects and registers bindex files in an external registry.
  • GWCommand – Processes documents/files using the Ghostwriter pipeline.
  • CleanCommand – Deletes Machai temporary directories (for example, .machai) under a selected root directory.

Typical usage

The following example illustrates a common workflow: configure defaults, generate/register metadata, discover libraries via semantic search, assemble an output project, process documentation, and finally clean temporary artifacts. 

config genai OpenAI:gpt-5.1
config dir .\\my-project
bindex --dir .\\my-project
pick --query "Create a web app" --score 0.9
assembly --dir .\\out
gw --dir .\\my-project --scan .\\my-project\\docs
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows.
  • Command names and options are defined by the Spring Shell components in org.machanism.machai.cli; consult each command type’s Javadoc for details.

Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Purpose and scope

Machai CLI produces an executable, Spring Shell-based command-line interface for Machai workflows. It supports interactive usage (REPL) as well as non-interactive invocation from scripts and CI pipelines.

Architecture at a glance

  • Entry point: org.machanism.machai.cli.MachaiCLI bootstraps the application, loads optional configuration from machai.properties (or a file specified via -Dconfig=...), and starts the Spring Shell runtime.
  • Command components: Spring Shell commands provide the user-facing operations (configuration, bindex generation/registration, semantic search and assembly, document processing, Act mode, and cleanup).

Package overview

The module’s public API surface is concentrated in a single package:

  • org.machanism.machai.cli – Spring Shell-based command-line interface (CLI) for Machai.
    • Contains the Spring Boot entry point and Spring Shell command components.
    • Supports both interactive REPL usage and one-shot command execution.
    • Command groups include:
      • Configuration – persists defaults (for example, working directory and GenAI provider/model).
      • Indexing and registration – generates and registers bindex metadata.
      • Picking and assembly – performs semantic search (“pick”) and assembles an output project.
      • Document processing – processes files via Ghostwriter; runs predefined interactive actions (“Act mode”).
      • Cleanup – removes Machai temporary artifacts (for example, .machai).

Key types and responsibilities

  • MachaiCLI – Application entry point; loads optional system properties and initializes Spring Shell.
  • ConfigCommand – Manages persisted defaults in machai.properties.
  • BindexCommand – Generates bindex metadata for projects and optionally registers bindex files.
  • AssembyCommand – Performs semantic search (“pick”) and assembles an output project using a configured GenAI provider.
  • GWCommand – Scans folders and processes documents/files using the Ghostwriter pipeline.
  • ActCommand – Runs a predefined interactive action/prompt (“Act mode”) against a selected working context.
  • CleanCommand – Removes Machai temporary directories (for example, .machai) under a selected root directory.

Typical usage

The following example illustrates a common workflow: configure defaults, generate/register metadata, discover libraries via semantic search, assemble an output project, process documentation, run an interactive action, and finally clean temporary artifacts. 

config model OpenAI:gpt-5.1
config root .\\my-project
bindex --dir .\\my-project
pick --query "Create a web app" --score 0.9
assembly --dir .\\out
gw --scanDirs .\\my-project --excludes target,.git
act --action "Summarize the architecture"
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows; cleanup is supported by the clean command.
  • Command names and options are defined by Spring Shell components in org.machanism.machai.cli; consult each command class’ Javadoc for details.

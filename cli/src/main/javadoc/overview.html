Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Purpose and scope

This module produces an executable, Spring Shell-based command-line interface for Machai workflows. It supports interactive usage (REPL) as well as non-interactive invocation from scripts and CI pipelines.

Architecture at a glance

  • Entry point: org.machanism.machai.cli.MachaiCLI bootstraps the application and starts the Spring Shell runtime.
  • Command components: Spring Shell commands implement the primary user-facing operations, including configuration, bindex generation/registration, semantic search (“pick”), project assembly, document processing (Ghostwriter), Act mode, and cleanup.
  • Configuration: defaults can be loaded from machai.properties (or from a file specified via -Dconfig=...) and are also manageable via the CLI.

Package overview

The public API surface for this module is concentrated in a single package:

  • org.machanism.machai.cli – Spring Shell-based command-line interface (CLI) for Machai.
    • Contains the Spring Boot entry point and Spring Shell command components.
    • Supports both interactive REPL usage and one-shot command execution.
    • Provides command groups for configuration, bindex operations, picking/assembly, document processing, Act mode, and cleanup.
    • Loads optional system properties from machai.properties or from a file referenced by -Dconfig=....

Key types and responsibilities

  • MachaiCLI – Application entry point; loads optional system properties and initializes Spring Shell.
  • ConfigCommand – Manages persistent defaults (for example, working directory, GenAI provider/model, and score threshold) stored in machai.properties.
  • BindexCommand – Generates bindex metadata for projects and optionally registers bindex files with an external registry.
  • AssembyCommand – Performs semantic search (“pick”) and assembles a project skeleton using a configured GenAI provider.
  • GWCommand – Scans folders and processes documents/files using the Ghostwriter pipeline.
  • ActCommand – Runs a predefined interactive action/prompt (“Act mode”) against a selected working context.
  • CleanCommand – Removes Machai temporary directories (for example, .machai) under a selected root directory.

Typical usage

The following example illustrates a common workflow: configure defaults, generate/register metadata, discover libraries via semantic search, assemble an output project, process documentation, run an interactive action, and finally clean temporary artifacts. 

config model OpenAI:gpt-5.1
config root .\\my-project
bindex --dir .\\my-project
pick --query "Create a web app" --score 0.9
assembly --dir .\\out
gw --scanDirs .\\my-project --excludes target,.git
act --action "Summarize the architecture"
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows. Clean-up is supported by the clean command.
  • Command names and options are defined by Spring Shell components in org.machanism.machai.cli; consult each command type’s Javadoc for details.

GW Maven Plugin

GW Maven Plugin is an advanced documentation automation tool for Java projects. It scans, analyzes, and assembles project documentation using embedded @guidance: tags and AI-powered synthesis. The plugin streamlines the creation and maintenance of high-quality, consistent documentation, helping keep documentation aligned with current source and site content across modules.

Overview

The plugin provides Maven goals (Mojos) that configure and invoke org.machanism.machai.gw.processor.GuidanceProcessor to traverse a configurable content tree (commonly src/site) and process files containing one or more @guidance: blocks. The processing model is intentionally content-type agnostic: it can operate on source code, documentation, Maven Site content, and other project files.

Packages

  • org.machanism.machai.gw.maven — Maven plugin goals and shared execution infrastructure. See the package documentation for details on configuration parameters and runtime behavior.

Goals

The primary goals exposed by this plugin are:

  • gw:gw (org.machanism.machai.gw.maven.GW) — Aggregator-style goal that can run without a pom.xml. It processes modules in reverse order (sub-modules first, then parent modules), similar to the Ghostwriter CLI.
  • gw:reactor (org.machanism.machai.gw.maven.ReactorGW) — Processes modules using standard Maven reactor dependency ordering, with an option to defer processing of the execution-root project.
  • gw:act (org.machanism.machai.gw.maven.Act) — Interactive goal for running predefined actions backed by resource bundles.
  • gw:clean (org.machanism.machai.gw.maven.Clean) — Deletes temporary artifacts created by GW processing (typically bound to Maven's clean lifecycle).

Scanning and processing model

At execution time, a goal determines the scan directory (often the Maven execution root or a user-supplied path) and applies include/exclude rules while walking the directory tree. Files containing @guidance: directives are passed to the Ghostwriter workflow engine for guided updates.

Configuration and credentials

In addition to standard Maven plugin parameters (for example, scan directory and exclude patterns), GenAI credentials can optionally be sourced from ~/.m2/settings.xml by providing -Dgw.genai.serverId=<serverId>. When configured, the plugin reads the matching <server> entry and forwards credentials to the workflow.

Common usage

Run guided processing for the current project: 

mvn gw:gw

Run reactor-ordered processing in a multi-module build: 

mvn gw:reactor

Override the scan root directory (Windows example): 

mvn gw:reactor -Dgw.scanDir=src\\site

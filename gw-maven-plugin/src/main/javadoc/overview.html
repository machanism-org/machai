GW Maven Plugin

GW Maven Plugin is a Maven plugin that integrates the MachAI Ghostwriter workflow into a Maven build. It scans a configurable directory tree (commonly src/site) for files that contain embedded @guidance: blocks and applies guided transformations using the configured GenAI provider.

Description (from POM): Ghostwriter Maven Plugin is an advanced documentation automation tool for Java projects. It automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis. The plugin streamlines the creation and maintenance of high-quality, consistent documentation, ensuring best practices and up-to-date information across all modules.

How guided processing works

  1. The goal determines a scan root (by default the execution root directory; optionally overridden via -Dgw.scanDir).
  2. The scan walks the directory tree, respecting any configured excludes.
  3. When a file contains a @guidance: block, the workflow composes an instruction set (goal parameters, default guidance, and the embedded guidance) and delegates the transformation to the configured GenAI provider.
  4. Temporary artifacts created by processing can be removed via gw:clean.

Package overview

org.machanism.machai.gw.maven

Maven plugin goals (Mojos) and supporting infrastructure for running the MachAI Ghostwriter (GW) guided document-processing workflow as part of a Maven build.

The goals in this package configure and invoke the Ghostwriter processors (for example, GuidanceProcessor and ActProcessor) to scan a documentation source tree for embedded @guidance: blocks.

Primary goals

  • gw:gw (org.machanism.machai.gw.maven.GW) — Aggregator-style goal that can run without a pom.xml. It processes modules in reverse order (sub-modules first, then parent modules), similar to the Ghostwriter CLI.
  • gw:reactor (org.machanism.machai.gw.maven.ReactorGW) — Processes modules using standard Maven reactor dependency ordering, with an option to defer processing of the execution-root project.
  • gw:act (org.machanism.machai.gw.maven.Act) — Interactive goal for running a predefined action/prompt over the scanned documents.
  • gw:act-reactor (org.machanism.machai.gw.maven.ReactorAct) — Reactor-friendly variant of gw:act intended to run in the execution-root project context.
  • gw:clean (org.machanism.machai.gw.maven.Clean) — Deletes temporary artifacts created by GW processing (typically bound to Maven's clean lifecycle).

Shared infrastructure

  • org.machanism.machai.gw.maven.AbstractGWGoal — Base class defining shared parameters (provider/model selection, instructions, default guidance, scan directory, excludes, optional credential lookup, and input logging) and the common scan/execute flow.

Configuration and credentials

In addition to standard Maven plugin parameters (for example, scan directory and exclude patterns), GenAI credentials can optionally be sourced from ~/.m2/settings.xml by providing -Dgw.genai.serverId=<serverId>. When configured, the plugin reads the matching <server> entry and forwards credentials to the workflow.

Common usage

Run guided processing

mvn gw:gw

Run reactor-ordered processing in a multi-module build

mvn gw:reactor

Override the scan root directory (Windows example)

mvn gw:reactor -Dgw.scanDir=src\\site

Use credentials from Maven settings

mvn gw:gw -Dgw.genai.serverId=my-genai-provider

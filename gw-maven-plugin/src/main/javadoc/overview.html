GW Maven Plugin

GW Maven Plugin is a Maven plugin that integrates the MachAI Ghostwriter guided document-processing workflow into a Maven build.

Description (from POM): Ghostwriter Maven Plugin is an advanced documentation automation tool for Java projects. It automatically scans, analyzes, and assembles project documentation using embedded guidance tags and AI-powered synthesis. The plugin streamlines the creation and maintenance of high-quality, consistent documentation, ensuring best practices and up-to-date information across all modules.

Core concepts

Guidance blocks
Embedded @guidance: sections inside project files that define requirements and constraints for content generation or transformation.
Processors
The plugin delegates to the MachAI Ghostwriter engine (for example, org.machanism.machai.gw.processor.GuidanceProcessor and org.machanism.machai.gw.processor.ActProcessor) to interpret guidance blocks and apply changes.
Scan root
The directory tree searched for files containing guidance blocks (commonly src/site), configurable via plugin parameters.

What the plugin does

  • Scans a configurable directory tree for files containing embedded @guidance: blocks.
  • Runs the configured Ghostwriter processor to apply guided, AI-assisted transformations through the selected GenAI provider.
  • Supports both single-module runs and multi-module runs, including reactor-ordered processing.

How guided processing works

  1. A goal selects a scan root (typically the Maven execution root, optionally overridden via a parameter such as -Dgw.scanDir).
  2. The scan walks the directory tree, respecting configured excludes.
  3. For each file containing a @guidance: block, the workflow composes instructions from goal parameters, default guidance, and embedded guidance.
  4. The configured GenAI provider is invoked to produce the transformed content, which is then written back to disk.
  5. Temporary artifacts created during processing can be removed via gw:clean.

Packages

org.machanism.machai.gw.maven

Maven plugin goals (Mojos) for running MachAI Ghostwriter (GW) guided document processing as part of Maven builds. The goals in this package configure and invoke org.machanism.machai.gw.processor.GuidanceProcessor (or specialized processors such as org.machanism.machai.gw.processor.ActProcessor) to scan a documentation source tree (commonly src/site) for files containing embedded @guidance: blocks.

When guidance blocks are found, the processor delegates content transformation to the configured GenAI provider. Provider credentials can optionally be sourced from ~/.m2/settings.xml by providing -Dgw.genai.serverId=<serverId>.

Main goals

  • gw:gw (org.machanism.machai.gw.maven.GW) — Aggregator goal intended to run across a multi-module build. It processes modules in reverse order (sub-modules first, then parent modules), similar to the Ghostwriter CLI.
  • gw:reactor (org.machanism.machai.gw.maven.ReactorGW) — Runs guided processing using standard Maven reactor dependency ordering, with an option to defer processing of the execution-root project.
  • gw:act (org.machanism.machai.gw.maven.Act) — Interactive goal for running a predefined action (act/prompt) over scanned documents.
  • gw:act-reactor (org.machanism.machai.gw.maven.ReactorAct) — Reactor-friendly variant of gw:act, intended to run in the execution-root project context.
  • gw:clean (org.machanism.machai.gw.maven.Clean) — Deletes temporary artifacts created by GW processing (typically bound to Maven's clean lifecycle).

Shared infrastructure

  • org.machanism.machai.gw.maven.AbstractGWGoal — Base class providing shared configuration and execution flow, including scan directory selection, excludes, optional credential lookup, and logging behavior.

Configuration and credentials

In addition to standard Maven plugin parameters, GenAI credentials can optionally be sourced from ~/.m2/settings.xml by providing -Dgw.genai.serverId=<serverId>. When configured, the plugin reads the matching <server> entry and forwards the credentials to the Ghostwriter engine.

Common usage

Run guided processing

mvn gw:gw

Run reactor-ordered processing in a multi-module build

mvn gw:reactor

Override the scan root directory (Windows example)

mvn gw:reactor -Dgw.scanDir=src\\site

Use credentials from Maven settings

mvn gw:gw -Dgw.genai.serverId=my-genai-provider

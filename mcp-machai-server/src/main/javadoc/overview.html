MCP Machai Server

Overview

MCP Machai Server is a server implementation for the Model Context Protocol (MCP) using the Machai AI framework. It provides integration with GenAI tools and supports both HTTP and STDIO interfaces for AI-powered tool execution and orchestration. The server is designed to be extensible, allowing seamless addition of new tools and capabilities for advanced AI-driven workflows.

Package Structure

  • Server Entry Points:
    • McpServer – Main entry point for launching the server in either STDIO or Remote (HTTP) mode, based on command-line options.
    • RemoteMcpServer – Configures and runs the MCP server over HTTP, supporting remote tool invocation.
    • StdioMcpServer – Configures and runs the MCP server over standard input/output for local integration.
  • Tool Integration:
    • GenericGenaiAdapter – Generic adapter for registering and managing GenAI tools, abstracting server-specific details.
    • ToolSpecificationBuilder, RemoteToolSpecificationBuilder, StdioToolSpecificationBuilder – Builders for tool specifications, enabling flexible tool registration for different server types.
  • Utilities:
    • Integration with FunctionToolsLoader and PropertiesConfigurator for dynamic tool loading and configuration.

Usage

To start the server, use the McpServer class with appropriate command-line options:

  • -h, --help: Show help message and exit.
  • -n, --name: Specify the MCP server name (default: mcp-machai-server).
  • -v, --version: Specify the MCP server version (default: implementation version or "latest").
  • -p, --port: Specify the port number for Remote MCP Server mode (HTTP).

If the --port option is provided, the server runs in remote (HTTP) mode; otherwise, it runs in STDIO mode for local integration.

Extensibility

The server is designed for extensibility. New tools can be registered using the GenericGenaiAdapter and appropriate ToolSpecificationBuilder implementations. This allows for rapid integration of new AI capabilities and custom workflows.

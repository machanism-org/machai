Assembly Maven Plugin

assembly-maven-plugin automates the assembly of projects within the Machanism ecosystem by integrating libraries based on bindex metadata. It streamlines dependency resolution, library selection, and project packaging using GenAI-powered semantic search and metadata-driven workflows.

What this plugin does

Assembly Maven Plugin is a Maven plugin (packaging type maven-plugin) that exposes an assembly goal. The goal uses a natural-language prompt to drive an AI-assisted workflow that identifies relevant libraries from bindex metadata and applies the resulting project changes directly to the Maven execution base directory (${basedir}).

High-level workflow

  1. Prompt acquisition: read the prompt from a configured file (for example project.txt) or collect it interactively.
  2. Library recommendation: use MachAI's picker to recommend candidate libraries described as bindex entries.
  3. Scoring and filtering: optionally discard candidates below a configured minimum score threshold.
  4. Assembly application: run the assembly engine to apply updates to the project directory in-place.

Package overview

org.machanism.machai.assembly.maven

Maven plugin entry point for MachAI's AI-assisted project assembly workflow. This package provides the assembly goal implementation and the minimal orchestration logic needed to:

  • obtain a natural-language prompt (file-based or interactive),
  • invoke MachAI's Picker to recommend libraries as bindex entries (optionally filtered by score), and
  • invoke the application assembly engine to apply changes to the target project directory.

The primary entry point is org.machanism.machai.assembly.maven.Assembly, a Maven Mojo that coordinates:

  • prompt input via the assembly.prompt.file setting (or interactive input),
  • recommendations produced by org.machanism.machai.bindex.Picker as org.machanism.machai.schema.Bindex entries, and
  • in-place changes applied by org.machanism.machai.bindex.ApplicationAssembly to ${basedir}.

Configuration and inputs

The goal can be configured via Maven properties and/or plugin configuration. Common parameters include:

  • assembly.genai – GenAI provider identifier used during the assembly phase.
  • pick.genai – GenAI provider identifier used during the recommendation (picker) phase.
  • assembly.prompt.file – text file containing the natural-language prompt.
  • assembly.score – minimum score threshold for recommendations.
  • bindex.register.url – optional endpoint used by the picker for metadata lookup/registration.

Typical usage

mvn org.machanism.machai:assembly-maven-plugin:assembly
  -Dassembly.genai=OpenAI:gpt-5
  -Dpick.genai=OpenAI:gpt-5-mini
  -Dassembly.prompt.file=project.txt
  -Dassembly.score=0.9

If assembly.prompt.file does not exist, the goal prompts for interactive input.

Operational notes

  • The plugin operates on the Maven execution base directory (${basedir}) and applies changes in-place.
  • Recommended libraries are produced from bindex metadata and are typically logged before the assembly phase begins.

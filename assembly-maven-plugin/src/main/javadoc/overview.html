Assembly Maven Plugin

assembly-maven-plugin automates the assembly of projects within the Machanism ecosystem by integrating libraries based on bindex metadata. It streamlines dependency resolution, library selection, and project packaging using GenAI-powered semantic search and metadata-driven workflows.

Packages

org.machanism.machai.assembly.maven

Maven plugin entry points for MachAI's AI-assisted project assembly workflow.

This package contains the Maven Mojo that exposes the assembly goal and orchestrates a full “prompt → recommend libraries → apply changes” pipeline:

  1. Acquire the prompt from a configured text file (typically project.txt) or by interactively asking the user.
  2. Recommend libraries by invoking MachAI’s picker component, which returns candidate libraries described as Bindex metadata entries.
  3. Filter recommendations using a minimum score threshold to keep only sufficiently relevant results.
  4. Assemble the project by applying the selected libraries to the target directory using the assembly engine.

Configuration and inputs

The goal is configured primarily via Maven properties (system properties) and/or plugin configuration. Common parameters include:

  • assembly.genai – GenAI provider identifier used during the assembly phase.
  • pick.genai – GenAI provider identifier used during the recommendation (picker) phase.
  • assembly.prompt.file – Text file containing the natural-language prompt.
  • assembly.score – Minimum score threshold for recommendations.
  • bindex.register.url – Optional endpoint used by the picker for metadata lookup/registration.

Typical usage

mvn org.machanism.machai:assembly-maven-plugin:assembly
  -Dassembly.genai=OpenAI:gpt-5
  -Dpick.genai=OpenAI:gpt-5-mini
  -Dassembly.prompt.file=project.txt
  -Dassembly.score=0.9

When assembly.prompt.file does not exist, the goal falls back to interactive prompt entry.

Notes

  • The plugin operates on the Maven execution base directory (${basedir}), applying changes in-place.
  • Recommended libraries are surfaced as bindex entries and may be logged before the assembly phase begins.

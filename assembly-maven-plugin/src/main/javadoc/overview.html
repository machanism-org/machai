Assembly Maven Plugin

assembly-maven-plugin automates the assembly of projects within the Machanism ecosystem by integrating libraries based on bindex metadata. It streamlines dependency resolution, library selection, and project packaging using GenAI-powered semantic search and metadata-driven workflows.

Purpose

Assembly Maven Plugin is a Maven plugin (packaging type maven-plugin) that exposes the assembly goal. The goal takes a natural-language prompt, uses MachAI's bindex metadata to recommend relevant libraries, filters recommendations by a configured minimum score threshold, and applies the selected libraries to the target project directory.

High-level workflow

  1. Prompt acquisition: read a prompt from a configured text file (typically project.txt) or request it interactively.
  2. Library recommendation: use the MachAI picker to recommend candidate libraries described as Bindex entries.
  3. Scoring and filtering: discard candidates below a configured minimum score threshold.
  4. Assembly application: apply the selected libraries to the Maven execution base directory (${basedir}) using the assembly engine.

Package overview

org.machanism.machai.assembly.maven

Maven plugin entry points for MachAI's AI-assisted project assembly workflow.

The main entry point is org.machanism.machai.assembly.maven.Assembly, a Maven {@code Mojo} that orchestrates:

  • prompt acquisition (from a configured prompt file or interactive input),
  • library recommendation via org.machanism.machai.bindex.Picker (as org.machanism.machai.schema.Bindex entries),
  • and in-place application of changes to ${basedir} via org.machanism.machai.bindex.ApplicationAssembly.

Configuration and inputs

The goal can be configured via Maven properties and/or plugin configuration. Common parameters include:

  • assembly.genai – GenAI provider identifier used during the assembly phase.
  • pick.genai – GenAI provider identifier used during the recommendation (picker) phase.
  • assembly.prompt.file – Text file containing the natural-language prompt.
  • assembly.score – Minimum score threshold for recommendations.
  • bindex.register.url – Optional endpoint used by the picker for metadata lookup/registration.

Typical usage

mvn org.machanism.machai:assembly-maven-plugin:assembly
  -Dassembly.genai=OpenAI:gpt-5
  -Dpick.genai=OpenAI:gpt-5-mini
  -Dassembly.prompt.file=project.txt
  -Dassembly.score=0.9

If assembly.prompt.file does not exist, the goal prompts for interactive input.

Operational notes

  • The plugin operates on the Maven execution base directory (${basedir}) and applies changes in-place.
  • Recommended libraries are surfaced as bindex entries and are typically logged before the assembly phase begins.

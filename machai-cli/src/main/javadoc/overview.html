Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Overview

This module provides a Spring Boot and Spring Shell based command-line interface (CLI) for running Machai workflows in an interactive REPL or in non-interactive mode suitable for scripts and CI pipelines. The CLI focuses on orchestrating higher-level operations (configuration, scanning, indexing, and cleanup) and delegates the heavy lifting to the underlying Machai libraries (for example, Ghostwriter and bindex).

Packages

  • org.machanism.machai.cli – application entry point and Spring Shell commands.
    • Provides command implementations and option parsing for common workflows.
    • Reads and persists CLI defaults (for example, working directory and provider/model selection).
    • Performs lightweight validation and path resolution, then invokes Machai services.

Primary components

  • org.machanism.machai.cli.MachaiCLI – application entry point.
  • org.machanism.machai.cli.GWCommand – scans folders and processes files with Ghostwriter.
  • org.machanism.machai.cli.ActCommand – runs Act-mode actions/prompts.
  • org.machanism.machai.cli.BindexCommand – generates (and optionally registers) bindex metadata.
  • org.machanism.machai.cli.ConfigCommand – reads/writes CLI configuration defaults.
  • org.machanism.machai.cli.CleanCommand – removes Machai temporary artifacts.

Key capabilities

  • Ghostwriter processing: scan a project and process selected files through the Ghostwriter pipeline.
  • Act mode: run predefined interactive actions/prompts in the context of a working directory.
  • Indexing and discovery: generate and optionally register bindex metadata for semantic search.
  • Picking and assembly: select libraries based on a query and assemble an output project.
  • Configuration: set and persist CLI defaults (for example, directory and provider/model settings).
  • Cleanup: remove Machai temporary artifacts (for example, .machai).

Typical usage

# Configure defaults
config set --key gw.model --value OpenAI:gpt-5.1
config dir .\\my-project

# Index a project for semantic search
bindex --dir .\\my-project

# Pick libraries and assemble an output project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out

# Run Ghostwriter processing and Act mode prompts
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"

# Clean temporary artifacts
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts produced by Machai workflows. Use the clean command to remove it when appropriate.
  • For command names, options, and default values, refer to the Javadoc on each command class in org.machanism.machai.cli.

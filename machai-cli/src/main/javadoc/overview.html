Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Module overview

This module provides a Spring Shell-based command-line interface (CLI) for the Machai toolchain. It supports both interactive usage (REPL) and non-interactive, one-shot execution suitable for automation and CI pipelines.

High-level architecture

  • Application entry point: org.machanism.machai.cli.MachaiCLI bootstraps the application and starts Spring Shell.
  • Command components: Spring Shell command classes expose user-facing operations such as configuration management, bindex generation/registration, semantic search (“pick”) and assembly, document processing (Ghostwriter and “Act mode”), and cleanup.

Package overview

Public functionality is concentrated in a single package:

  • org.machanism.machai.cli – Spring Shell-based CLI for Machai.
    • Contains the Spring Boot entry point and Spring Shell command components that implement Machai workflows.
    • Commands are designed to work well in both interactive and scripted contexts.
    • Primary command groups include configuration, indexing/registration, picking/assembly, document processing, and cleanup.

Key types

  • org.machanism.machai.cli.MachaiCLI – Application entry point; initializes Spring Shell.
  • org.machanism.machai.cli.ConfigCommand – Persists and prints defaults (for example, root directory and GenAI model).
  • org.machanism.machai.cli.BindexCommand – Generates and registers bindex metadata for projects.
  • org.machanism.machai.cli.AssembyCommand – Performs semantic search (“pick”) and assembles an output project.
  • org.machanism.machai.cli.GWCommand – Scans folders and processes documents/files using the Ghostwriter pipeline.
  • org.machanism.machai.cli.ActCommand – Runs a predefined interactive action/prompt (“Act mode”).
  • org.machanism.machai.cli.CleanCommand – Removes Machai temporary artifacts (for example, .machai).

Typical usage

config genai OpenAI:gpt-5.1
config dir .\\my-project
bindex --dir .\\my-project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows. Use the clean command to remove it when needed.
  • Command names and options are defined by Spring Shell components in org.machanism.machai.cli. Consult each command class’ Javadoc for detailed semantics and option defaults.

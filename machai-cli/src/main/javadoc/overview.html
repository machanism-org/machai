Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Purpose

This module provides a Spring Shell-based command-line interface for the Machai toolchain. It is designed for:

  • Interactive use via a REPL-style shell.
  • Non-interactive execution suitable for scripting and CI automation.
  • Delegation to core Machai libraries (for example, bindex and Ghostwriter) to perform indexing, semantic search, and guided file processing.

High-level architecture

  • Application bootstrap: org.machanism.machai.cli.MachaiCLI starts the JVM process and initializes Spring Shell.
  • Command layer: Command classes in org.machanism.machai.cli expose user-facing operations, map shell options to typed parameters, and coordinate console I/O.
  • Execution & delegation: Commands validate inputs, resolve working directories, read defaults from configuration, and then delegate work to underlying Machai modules (for indexing, search, prompt execution, and assembly).

Package overview

Public CLI functionality is concentrated in the following package:

  • org.machanism.machai.cli – Command-line interface (CLI) entry points and supporting components.
    • Defines Spring Shell commands that expose Machai workflows as interactive or scriptable operations.
    • Coordinates option parsing, console output, and basic validation before invoking Machai libraries.
    • Uses persisted configuration (via ConfigCommand) to provide defaults such as project directory and model/provider settings.

Primary command components

  • org.machanism.machai.cli.MachaiCLI – Application entry point; initializes Spring Shell.
  • org.machanism.machai.cli.ConfigCommand – Reads/writes CLI defaults and configuration.
  • org.machanism.machai.cli.BindexCommand – Generates (and optionally registers) bindex metadata.
  • org.machanism.machai.cli.AssembyCommand – Performs semantic search selection and assembles an output project.
  • org.machanism.machai.cli.GWCommand – Scans folders and processes files using the Ghostwriter pipeline.
  • org.machanism.machai.cli.ActCommand – Runs predefined interactive actions/prompts (Act mode).
  • org.machanism.machai.cli.CleanCommand – Removes Machai temporary artifacts (for example, .machai).

Typical usage

# Configure defaults
config set --key gw.model --value OpenAI:gpt-5.1
config dir .\\my-project

# Index a project for semantic search
bindex --dir .\\my-project

# Pick libraries and assemble an output project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out

# Run Ghostwriter processing and Act mode prompts
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"

# Clean temporary artifacts
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts produced by Machai workflows. Use the clean command to remove it when appropriate.
  • For command names, options, and default values, refer to the Javadoc on each command class in org.machanism.machai.cli.

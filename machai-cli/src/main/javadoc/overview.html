Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Module overview

This module provides a Spring Shell-based command-line interface for the Machai toolchain. It supports interactive execution (REPL) as well as non-interactive usage suitable for scripts and CI pipelines.

Key capabilities

  • Guided file processing: scan a project and process files using the Ghostwriter pipeline.
  • Act mode: run predefined interactive actions/prompts against a working directory.
  • Indexing and discovery: generate and optionally register bindex metadata for semantic search.
  • Picking and assembly: select libraries based on a query and assemble an output project.
  • Configuration: persist CLI defaults (for example, project directory and model/provider settings).
  • Cleanup: remove Machai temporary artifacts (for example, .machai).

Architecture

  • Application entry point - org.machanism.machai.cli.MachaiCLI bootstraps the process and initializes Spring Shell.
  • Command layer - classes in org.machanism.machai.cli expose user-facing operations, map shell options to typed parameters, and coordinate console I/O.
  • Delegation to core libraries - commands perform basic validation and directory resolution and then delegate to underlying Machai modules (for example, bindex and Ghostwriter) to execute the workflow.

Package overview

The primary package in this module is:

  • org.machanism.machai.cli - Spring Shell entry point and CLI command implementations.
    • Orchestrates Machai workflows such as scanning, Act-mode execution, bindex generation, and cleanup.
    • Provides persistent configuration used as defaults by most commands.
    • Serves as the primary integration point for scripting and CI automation.

Primary components

  • org.machanism.machai.cli.MachaiCLI - application entry point.
  • org.machanism.machai.cli.ConfigCommand - reads/writes CLI configuration defaults.
  • org.machanism.machai.cli.GWCommand - scans folders and processes files with Ghostwriter.
  • org.machanism.machai.cli.ActCommand - runs predefined interactive actions/prompts (Act mode).
  • org.machanism.machai.cli.BindexCommand - generates (and optionally registers) bindex metadata.
  • org.machanism.machai.cli.AssembyCommand - performs pick/assembly workflows based on semantic search.
  • org.machanism.machai.cli.CleanCommand - removes Machai temporary artifacts.

Typical usage

# Configure defaults
config set --key gw.model --value OpenAI:gpt-5.1
config dir .\\my-project

# Index a project for semantic search
bindex --dir .\\my-project

# Pick libraries and assemble an output project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out

# Run Ghostwriter processing and Act mode prompts
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"

# Clean temporary artifacts
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts produced by Machai workflows. Use the clean command to remove it when appropriate.
  • For command names, options, and default values, refer to the Javadoc on each command class in org.machanism.machai.cli.

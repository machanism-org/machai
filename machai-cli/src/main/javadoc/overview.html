Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

What this module provides

This module is a Spring Shell-based command-line interface for the Machai toolchain. It is intended for both interactive usage (REPL) and non-interactive execution suitable for automation and CI pipelines.

High-level architecture

  • Bootstrap: org.machanism.machai.cli.MachaiCLI starts the CLI and wires Spring Shell.
  • Commands: Spring Shell components expose user-facing operations such as configuration management, bindex generation/registration, semantic search and assembly, Ghostwriter processing, and cleanup.
  • Delegation: Command implementations call into Machai libraries (for example bindex-core and ghostwriter) to perform indexing, semantic search, prompt execution, and project assembly.

Package overview

Public CLI functionality is concentrated in:

  • org.machanism.machai.cli – Command-line interface (CLI) layer for Machai.
    • Contains Spring Shell commands that expose Machai workflows as interactive or scriptable CLI operations.
    • Commands typically read defaults from persisted configuration managed by org.machanism.machai.cli.ConfigCommand, resolve a working/project directory, and delegate execution to the underlying Machai modules.
    • Typical operations include running Ghostwriter guidance processing, creating and registering a bindex, performing semantic search (pick), assembling a project from selected libraries, and removing temporary artifacts.

Primary command types

  • org.machanism.machai.cli.MachaiCLI – Application entry point; initializes Spring Shell.
  • org.machanism.machai.cli.ConfigCommand – Reads/writes CLI defaults (for example, project directory and GenAI model).
  • org.machanism.machai.cli.BindexCommand – Generates and optionally registers bindex metadata for projects.
  • org.machanism.machai.cli.AssembyCommand – Performs semantic search (pick) and assembles an output project.
  • org.machanism.machai.cli.GWCommand – Scans folders and processes files using the Ghostwriter pipeline.
  • org.machanism.machai.cli.ActCommand – Runs a predefined interactive action/prompt ("Act mode").
  • org.machanism.machai.cli.CleanCommand – Removes Machai temporary artifacts (for example, .machai).

Typical usage

# Interactive or script-driven command execution
config set --key gw.model --value OpenAI:gpt-5.1
config dir .\\my-project

# Index a project for semantic search
bindex --dir .\\my-project

# Pick libraries and assemble an output project
pick --query "Create a web app" --score 0.8
assembly --dir .\\out

# Run Ghostwriter processing and Act mode prompts
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"

# Clean temporary artifacts
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts generated by Machai workflows. Use the clean command to remove it when needed.
  • Command names and options are defined by Spring Shell components in org.machanism.machai.cli. Refer to each command class' Javadoc for option defaults and semantics.

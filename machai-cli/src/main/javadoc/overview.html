Machai CLI

Machai CLI is a command-line tool for generating, registering, and managing library metadata within the Machanism ecosystem. It leverages GenAI to automate project assembly and enable semantic search for efficient library discovery and integration.

Module overview

This module provides a Spring Boot / Spring Shell command-line interface that can be used interactively (REPL) or non-interactively (scripts and CI). The CLI primarily orchestrates workflows: parsing options, resolving paths, applying persisted defaults, and delegating the core work to the underlying Machai libraries.

Package summary

The public API surface of this module is concentrated in a single package:

  • org.machanism.machai.cli – Command-line interface (CLI) commands and application entry point for Machai.
    • Provides the Spring Boot / Spring Shell wiring required to run Machai workflows from a terminal.
    • Exposes commands for configuration management, Ghostwriter processing, Act-mode execution, and cleanup.
    • Supports both interactive sessions (REPL) and non-interactive execution (scripts/CI).

Primary types

  • org.machanism.machai.cli.MachaiCLI – application entry point.
  • org.machanism.machai.cli.GWCommand – Ghostwriter scanning and processing orchestration.
  • org.machanism.machai.cli.ActCommand – Act-mode prompt execution.
  • org.machanism.machai.cli.ConfigCommand – persistent CLI configuration management.
  • org.machanism.machai.cli.CleanCommand – cleanup of Machai temporary folders.

Key capabilities

  • Configuration management: set and persist defaults used by other commands.
  • Ghostwriter orchestration: scan folders and run guidance-driven processing on selected files.
  • Act-mode execution: run predefined actions/prompts against a working directory context.
  • Cleanup: remove temporary artifacts created by Machai workflows (for example, .machai).

Typical usage

# Configure defaults
config set --key gw.model --value OpenAI:gpt-5.1
config dir .\\my-project

# Run Ghostwriter processing and Act mode prompts
gw --scanDir .\\my-project --excludes target,.git
act --action "Summarize the architecture"

# Clean temporary artifacts
clean --dir .\\my-project

Operational notes

  • The .machai directory is used for temporary artifacts produced by Machai workflows; remove it using clean when appropriate.
  • For command names and options, refer to the Javadoc on the command classes in org.machanism.machai.cli.
